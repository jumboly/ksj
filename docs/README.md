# KSJ 統合ツール 仕様書

国土数値情報 (KSJ, 国土交通省) のカタログ管理・ダウンロード・統合 CLI ツール `ksj` の仕様。

## 背景

KSJ のデータ配布単位は大きく 3 パターンある:

1. 全国一括
2. 都道府県単位 (47 分割)
3. 標準地域メッシュ単位 (1〜6 次メッシュ)

都道府県・メッシュ分割のデータには全国版が存在しないため、利用者は自前で結合する必要がある。さらに以下の事情が扱いを難しくしている:

- 測地系の混在 (旧測地系 / JGD2000 / JGD2011 / WGS84)
- 文字コード (Shift_JIS / UTF-8)
- 年次改版によるスキーマ差異
- URL パターンのデータセット間非一貫性 (ホストが異なる、年号表記が YYYYMMDD / YY 混在、圏域コードが文字列 SYUTO/CHUBU/KINKI 等)
- 形式の並列配布 (同じ年・同じ分割単位で Shapefile / GML / GeoJSON / CityGML / CSV / GeoTIFF が共存)

本ツールは KSJ のカタログ管理・ダウンロード・統合・形式変換までを再現可能な形で CLI で実行できるようにする。CLI で価値を出すことを最初の目標とし、TUI / Web UI の拡張余地は残す。

## 前提決定事項

| 項目 | 決定内容 |
|---|---|
| 言語 | Python 3.12+ |
| パッケージ管理 | uv (`uv run ksj ...` 実行) |
| UI | CLI が最優先、TUI/Web は将来拡張 |
| カタログ | リポジトリ同梱 YAML + `ksj catalog refresh` で再生成 |
| データ保存 | プロジェクト内 `./data/` (デフォルト)、`--data-dir` で上書き |
| 出力形式 | GeoPackage |
| 対象範囲 | KSJ 全データセットのカタログ化、統合検証は MVP 5 件 |
| 統合レベル | 結合 + CRS 統一 + 属性正規化 (スキーマ統一 + UTF-8) |
| デフォルト CRS | JGD2011 (EPSG:6668)、`--target-crs` で上書き |
| バージョン管理 | `--year` で明示指定、複数年次を並存 |
| ダウンロード | HTTP 並列取得、ホスト別レート制限、Range レジューム、ローカル ZIP 取込 |
| UX | rich による進捗バー・色付きログ・テーブル表示 |
| メタデータ | 統合ファイル内に出典・ライセンス・生成日を埋込 + 別途 manifest.json |
| 品質保証 | pytest, ruff, mypy (strict), 実データ E2E |

## 非目標 (スコープ外)

- メッシュ境界における重複ポリゴンのマージやトポロジ修正
- 属性フィールド名の英語化
- TUI / Web UI (将来拡張)
- SHA256 チェックサム照合 (KSJ 側未提供のため、取得時の値を自前で記録するのみ)
- ラスタデータ (L03-b_r のみ該当) の統合
- CSV 専有データ (L01/L02 地価公示) の座標フィールド解釈 — 将来フェーズで CSV→ポイント変換を実装
- TKY2JGD / PatchJGD による高精度測地系変換
- 部分カバレッジデータの「全国相当化」フィリング (未整備県への代替データ補完はしない)
